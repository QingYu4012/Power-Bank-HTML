## 2. 项目概览

### 2.1 项目定位

这是一个**纯前端、单页面、零后端依赖**的 Web 串口调试助手，核心面向“串口透传调试”场景。

它通过浏览器的 `Web Serial API` 直接与串口设备通信，提供可视化界面完成：

1. 串口授权与连接
2. 参数配置（波特率等）
3. 字符串/HEX 数据发送
4. 实时接收数据显示
5. 循环发送
6. 文件分片发送
7. 日志查看、清空、导出

### 2.2 技术栈与结构

项目非常集中，只有两个核心文件：

1. `index.html`：页面结构 + CSS 样式 + JavaScript 逻辑（全部在一个文件内）
2. `README.md`：项目说明、版本变更、部署说明

技术实现：

1. HTML5：页面布局与控件
2. CSS3：界面样式、响应式布局、日志区域视觉
3. 原生 JavaScript：串口通信、状态机、事件绑定、文件传输、日志管理
4. 浏览器能力：`navigator.serial`（Web Serial API）、`Blob` 下载、`TextEncoder/TextDecoder`

### 2.3 运行环境要求

1. 浏览器必须支持 Web Serial（建议新版 Chrome/Edge）
2. 必须在安全上下文下访问：
   - `https://` 域名
   - 或 `http://localhost`
3. 不需要后端服务器，静态托管即可（例如 GitHub Pages）

## 3. 页面模块与功能实现说明

页面分为左侧“控制区”和右侧“日志区”，每个区域承担不同职责。

### 3.1 串口连接模块

功能入口：

1. `选择串口`
2. `刷新列表`
3. `已授权端口` 下拉框
4. `连接` / `断开`
5. 参数输入区（波特率、数据位、停止位、校验位、流控）
6. 顶部连接状态胶囊（未连接/已连接/错误）

实现逻辑：

1. 通过 `navigator.serial.requestPort()` 请求用户授权一个设备
2. 通过 `navigator.serial.getPorts()` 获取已授权设备列表
3. 将端口显示为“串口 n (VID:xxxx PID:xxxx)”格式，便于识别硬件
4. 连接时读取并提交参数：
   - 波特率：默认 `115200`
   - 数据位：默认 `8`
   - 停止位：默认 `1`
   - 校验位：默认 `none`
   - 流控：默认 `none`，支持 `hardware (RTS/CTS)`
5. 连接成功后：
   - 建立 writer（写通道）
   - 启动读取循环（读通道）
   - 更新按钮可用状态
   - 写入系统日志
6. 断开时执行完整清理：
   - 停止读取循环
   - 停止循环发送
   - 取消文件发送（若正在进行）
   - 释放 reader/writer 锁
   - 关闭端口
   - 重置状态和控件

### 3.2 发送控制模块（文本/HEX）

功能入口：

1. 发送模式：`字符串` / `HEX`
2. 自动换行：`不追加` / `LF` / `CRLF` / `CR`
3. 发送内容输入框
4. `发送` 按钮
5. `清空输入`
6. 快捷键：`Ctrl+Enter`（Mac 下支持 `Command+Enter`）

实现逻辑：

1. 字符串模式：
   - 输入内容 + 行尾控制（可选）后，使用 `TextEncoder` 编码为字节
2. HEX 模式：
   - 支持输入 `0x` 前缀、空格、换行、逗号作为分隔
   - 自动清理分隔符后校验：
     - 只能包含 `0-9/A-F`
     - 长度必须为偶数（2位一个字节）
   - 转换为字节数组后可拼接行尾字节
3. 所有发送最终通过 `queueWrite()` 串行队列写入，避免并发写冲突
4. 发送成功写 `TX` 日志，失败写 `ERR` 日志

### 3.3 实时接收模块

功能入口：

1. 接收显示模式：`字符串` / `HEX` / `字符串 + HEX`

实现逻辑：

1. 连接后进入 `startReadLoop()` 持续读取串口数据
2. 每次收到字节数据执行显示转换：
   - 字符串模式：`TextDecoder` 解码，`\r\n\t` 等控制字符转义显示
   - HEX 模式：按字节转大写十六进制
   - 双视图模式：同一条日志内同时显示文本与 HEX
3. 数据记录为 `RX` 日志，并支持自动滚动到最新行

### 3.4 循环发送模块

功能入口：

1. 循环周期（ms，最小 10ms，默认 1000ms）
2. `启动循环发送`
3. `停止循环发送`

实现逻辑：

1. 启动时校验已连接串口
2. 使用 `setTimeout` 周期调度发送任务
3. 使用 `loopBusy` 防重入，避免前一次发送未完成时重叠执行
4. 停止时清理定时器并恢复控件状态
5. 所有动作记录 `SYS`/`ERR` 日志

### 3.5 文件发送模块

功能入口：

1. 选择文件
2. 分片大小（字节，默认 `512`）
3. 分片间隔（ms，默认 `0`）
4. `开始发送文件`
5. `取消发送`
6. 进度条 + 百分比/字节数

实现逻辑：

1. 读取本地文件为 `ArrayBuffer` 并转 `Uint8Array`
2. 按 `chunkSize` 分片发送，每片写入走 `queueWrite()`
3. 每片后更新进度 UI
4. 当 `delay > 0` 时，在分片间插入等待，减轻设备接收压力
5. 用户点击取消时，将 `abortFileSend=true`，循环自然退出
6. 完成/取消/失败均写入日志

### 3.6 日志系统模块

功能入口：

1. 日志区滚动展示
2. 自动滚动开关
3. `清空日志`
4. `保存日志`
5. 日志计数显示

日志类型：

1. `RX`：接收数据
2. `TX`：发送数据
3. `SYS`：系统提示（连接、状态变更等）
4. `ERR`：错误信息

实现细节：

1. 每条日志自动带毫秒时间戳 `[HH:mm:ss.SSS]`
2. 最大保留 `5000` 条，超限自动移除最旧日志（内存与UI双清理）
3. 保存日志时生成本地文本文件：
   - 文件名示例：`serial-log-20260419-153045.txt`
   - 内容为当前日志快照（CRLF 换行）
4. 日志文件在浏览器本地生成下载，不上传服务器

### 3.7 设备热插拔与稳定性处理

实现逻辑：

1. 监听 `navigator.serial` 的 `connect` 事件：
   - 新设备接入时刷新端口列表
2. 监听 `disconnect` 事件：
   - 若断开的是当前已连接端口，自动执行断连清理并记录错误
   - 若断开的是其他端口，仅刷新列表并提示
3. 这使得工具在“设备被拔线”的真实调试场景下更稳定

### 3.8 安全上下文提示

实现逻辑：

1. 页面启动时检查 `window.isSecureContext` 或 `localhost/127.0.0.1`
2. 若不满足条件，顶部显示显著提示 Banner
3. 避免用户在不支持 Web Serial 的访问方式下误操作

### 3.9 响应式界面能力

1. 大屏默认双栏布局（左控制、右日志）
2. 在 `1100px` 以下切换为单栏
3. 在 `560px` 以下进一步优化：
   - 缩小页边距
   - 参数网格改单列
   - 按钮全宽，便于触屏操作

## 4. 核心状态机（帮助理解系统行为）

### 4.1 连接状态

1. 未连接：可授权、可连接，不可发送
2. 已连接：可发送、可循环、可文件发送，可断开
3. 错误：连接失败或运行错误，日志显示原因

### 4.2 发送状态

1. 普通发送：按点击触发或快捷键触发
2. 循环发送：`loopEnabled=true` 后按周期持续发送
3. 文件发送：`fileSending=true` 时禁用重复启动，支持取消

### 4.3 互斥与并发控制

1. `writeQueue` 保证串口写入串行化
2. `loopBusy` 防止循环发送重入
3. 文件发送与取消通过 `abortFileSend` 协作退出

## 5. 用户操作手册（一步一步）

以下流程以“第一次使用”作为基线。

### 5.1 启动前准备

1. 使用 Chrome 或 Edge 新版本
2. 确保页面访问地址为 `https://...` 或 `http://localhost`
3. 将待调试串口设备连接到电脑

### 5.2 基础连接流程

1. 打开页面，确认右上状态为“未连接”
2. 点击 `选择串口`，在浏览器弹窗里选中目标设备并确认授权
3. 如未自动出现，点击 `刷新列表`
4. 在“已授权端口”下拉框里选择目标端口
5. 配置串口参数（不确定时先用默认值）
6. 点击 `连接`
7. 日志出现连接成功信息后即可开始收发

### 5.3 文本发送流程

1. 在“发送模式”选择 `字符串`
2. 根据协议选择是否追加行尾（常见设备需要 `CRLF` 或 `LF`）
3. 在输入框填写命令，如 `AT`、`help` 等
4. 点击 `发送` 或按 `Ctrl+Enter`
5. 在日志区查看 `TX` 与设备 `RX` 反馈

### 5.4 HEX 发送流程

1. 在“发送模式”选择 `HEX`
2. 输入十六进制字节串，例如 `01 03 00 00 00 02 C4 0B`
3. 可混用空格、换行、逗号，也可写 `0x01 0x03 ...`
4. 点击 `发送`
5. 若输入非法字符或奇数长度，界面会给出 `ERR` 日志

### 5.5 实时接收查看

1. 在日志栏上方选择接收显示方式：
   - 仅字符串：适合文本协议
   - 仅 HEX：适合二进制协议
   - 字符串 + HEX：适合排查编码问题
2. 保持“自动滚动”勾选可持续跟踪最新数据

### 5.6 循环发送流程

1. 在输入框准备好要周期发送的内容
2. 设置“循环周期（ms）”，例如 `1000`
3. 点击 `启动循环发送`
4. 观察日志持续输出
5. 需要停止时点击 `停止循环发送`

建议：

1. 周期不要过小（尤其设备性能较弱时）
2. 若设备回包变慢，适当增大周期

### 5.7 文件发送流程

1. 点击“选择文件”选中待发送文件
2. 设置“分片大小”：
   - 小分片更稳（如 128/256/512）
   - 大分片速度更快但对设备缓存要求更高
3. 设置“分片间隔”：
   - `0ms` 追求速度
   - `1~20ms` 常用于防止设备来不及处理
4. 点击 `开始发送文件`
5. 观察进度条和日志
6. 需要中断时点击 `取消发送`

### 5.8 日志管理流程

1. 点击 `清空日志` 可重置当前会话视图
2. 点击 `保存日志` 可导出本次调试记录
3. 导出的文本可用于问题追踪、测试留档、对外协作

### 5.9 断开流程

1. 调试完成后点击 `断开`
2. 如设备已被拔掉，系统会自动检测并清理连接状态

## 7. 常见问题与排查

### 7.1 页面提示浏览器不支持

原因：

1. 浏览器版本过旧
2. 非 Chromium 内核浏览器

处理：

1. 使用最新 Chrome/Edge

### 7.2 选择串口后下拉框没有设备

原因：

1. 用户在授权弹窗中取消了选择
2. 设备驱动或连接异常
3. 授权与当前页面来源不一致

处理：

1. 重新点击 `选择串口`
2. 点击 `刷新列表`
3. 重新插拔设备并观察系统设备管理器

### 7.3 能连接但收不到数据

原因：

1. 串口参数不匹配（波特率/校验位等）
2. 设备协议要求特定换行符
3. 设备需要先发送“唤醒命令”

处理：

1. 核对参数
2. 尝试切换 `LF/CRLF/CR`
3. 切换接收显示为 `字符串 + HEX` 辅助定位

### 7.4 HEX 发送报错

原因：

1. 输入非十六进制字符
2. 字节串长度为奇数

处理：

1. 仅保留 `0-9A-F`
2. 保证每个字节两位，例如 `0A` 而不是 `A`

### 7.5 文件发送中断或失败

原因：

1. 设备处理中断开
2. 分片太大或发送过快

处理：

1. 减小分片大小
2. 增加分片间隔
3. 重新连接后再次发送

### 7.6 页面显示“需要 HTTPS 或 localhost”

原因：

1. 你正在使用不安全上下文（如本地文件直接双击、普通 HTTP 站点）

处理：

1. 改为 `http://localhost` 或部署到 `https://` 站点
